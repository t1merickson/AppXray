## Findings not yet planned (recorded for a future session)

- **DEBT-04 — Technology metadata is ~7 lockstep sites.** Adding one flag means
  touching the OptionSet constant, `allKnown`, `displayName`, `symbolName`, a
  category array (`DetectedTechnologies.swift`), plus `SummaryView` texts and
  the README table; a missed `displayName` silently drops the row from the UI
  (`items(in:)` skips nil names). Fix: a single static registry
  `[(flag, name, symbol, category)]` the other lists derive from, plus an
  exhaustiveness test. Effort M, risk LOW-MED. **Do after 001** (tests catch
  refactor slips). High value if detection keeps growing.
- **BUG-05 — `InfoDict.B()` treats plist string `"0"` as true**
  (`InfoDict.swift:46-53`) — ancient apps with string-typed `LSRequiresCarbon`
  misreport Carbon. S effort, low real-world frequency.
- **BUG-06 — Drop-path URL delivery still uses `RunLoop.main.perform`**
  (`URLItems.swift:30`) while the pipeline deliberately moved to
  `DispatchQueue.main.async` for run-loop-mode reasons
  (`BundleFeatureDetectionOperation.swift:83-85`). Consistency fix, S.
- **ROBUST-07 — Watchdog data race in `ProcessHelper.swift:97-112`**:
  `timedOut` written on a global queue, read unsynchronized after
  `waitUntilExit()`. Worst case a timed-out scan reports a normal exit status;
  TSan would flag it. S.
  (05/06/07 bundle naturally into one small "correctness sweep" plan.)

## Findings considered and rejected

- **`extractPrintableStrings` performance** (byte-wise String building +
  repeated `contains` scans): bounded at 4 MB, worst case a few hundred ms per
  binary — not worth the complexity of a byte-level matcher.
- **Parallelizing the transitive objdump walk**: after plan 003 removes the
  phantom spawns, the remaining sequential walk is small; complexity unjustified.
- **objdump `--rpaths` second pass**: already considered and resolved via
  static-marker alternative — recorded in `AUDIT.md`.
- **Raising `binaryScanLimit` (4 MB prefix)**: documented, deliberate
  speed/coverage trade-off (`README.md` "How it works" §4); tuning it needs
  evidence of a real missed detection first.
- **Security posture**: sandboxed, read-only user-selected access, no-shell
  subprocess default with quoting fallback, timeout watchdog, bounded reads —
  audited, nothing to report.
- **Docs**: both READMEs are current, accurate, and unusually good; `AUDIT.md`
  status header matches the code. Nothing stale found.
- **`ContentView.loadURLs` using `Optional.flatMap` for side effects**:
  style-only, zero behavioral cost.
